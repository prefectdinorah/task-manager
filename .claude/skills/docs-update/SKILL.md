# Documentation Update Skill

**Команда:** `/docs-update`

**Назначение:** Обновление всей проектной документации после изменений.

## Что делает эта команда

1. **Обновляет changelog.md** - добавляет новую запись об изменениях
2. **Обновляет project_status.md** - актуализирует текущий статус проекта
3. **Обновляет architecture.md** - если изменилась архитектура
4. **Создаёт/обновляет feature спецификации** в `docs/features/`

## Процесс работы

### 1. Сбор информации
- Читает последние коммиты из git
- Анализирует изменённые файлы
- Проверяет текущую документацию

### 2. Обновление changelog.md
Формат записи:
```markdown
## [Дата] - Версия/Описание

### Added
- Новые фичи

### Changed
- Изменения существующих фич

### Fixed
- Исправления багов

### Removed
- Удалённый функционал
```

### 3. Обновление project_status.md
- Текущая версия
- Статус разработки
- Последние изменения
- Планы на будущее

### 4. Обновление architecture.md
Только если были изменения в:
- Структуре проекта
- Компонентах
- API endpoints
- База данных схеме
- Инфраструктуре

### 5. Feature спецификации
- Создаёт новые спецификации для новых фич
- Обновляет существующие если изменились требования

## Использование

```bash
# Простой вызов - автоматическое обновление
/docs-update

# Или через npm скрипт
npm run docs:update
```

## Примеры

### После добавления новой фичи
```
/docs-update
```

Claude:
1. Читает коммиты
2. Находит новую фичу (feat: добавлена авторизация)
3. Обновляет changelog.md с записью о новой фиче
4. Обновляет project_status.md
5. Создаёт docs/features/authorization.md
6. Проверяет нужно ли обновить architecture.md

### После hotfix
```
/docs-update
```

Claude:
1. Читает hotfix коммиты
2. Обновляет changelog.md в секции Fixed
3. Обновляет project_status.md
4. НЕ создаёт новую feature спецификацию

## ВАЖНО

- ❌ НЕ коммитить .md файлы (они в .gitignore)
- ✅ Все изменения в документации остаются локально
- ✅ Владелец сам решает что коммитить
- ✅ Документация всегда актуальна для локальной работы

## Структура docs/

```
docs/
├── project_spec.md       # Общее описание проекта
├── architecture.md       # Архитектура системы
├── changelog.md          # История изменений
├── project_status.md     # Текущий статус
└── features/             # Спецификации фич
    ├── feature-1.md
    ├── feature-2.md
    └── ...
```
